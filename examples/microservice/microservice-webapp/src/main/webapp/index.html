Microservice Architecture Demo

This application demonstrates integration between a JAR library and a WAR webapp in a Tomcat environment.

Architecture:

┌─────────────────────────────────────┐
│           WAR Application           │
│  ┌─────────────────────────────────┐│
│  │      MicroserviceServlet        ││
│  │    (Web Layer/HTTP Handler)     ││
│  └─────────────┬───────────────────┘│
│                │ imports              │
│  ┌─────────────▼───────────────────┐│
│  │      microservice-lib.jar       ││
│  │   (Business Logic Library)      ││
│  │   - MicroserviceLib             ││
│  │   - getGreeting()               ││
│  │   - getVersion()                ││
│  │   - getServiceInfo()            ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘

How It Works:

  1. JAR Library: Contains pure business logic (MicroserviceLib)
  2. WAR Webapp: Contains servlet that exposes logic via HTTP
  3. Integration: JAR is included in WEB-INF/lib/ of the WAR
  4. Deploy: Tomcat loads WAR and automatically includes JAR
  5. Request: HTTP GET → Servlet → JAR Library → JSON Response

Benefits of This Architecture:

  • Separation of concerns: Business logic separate from web logic
  • Reusability: JAR library can be used in other projects
  • Testing: Business logic testable independently from web
  • Maintainability: Logic changes don't impact web layer
  • Deployment: Single WAR contains everything needed
